# AI 学术会议论文起草Prompt

## 1.1 论文结构拆解

```Markdown
你是==xxx==领域的博士生导师，有丰富的研究和投稿经验，多次在==xxx==等一系列高级学术会议中发表论文。你精通指导学生完成研究工作和投稿论文。以下是学生对他研究工作的简介，你需要认真阅读并分析，总结出他在论文写作时需要确认的一系列问题：
1、该工作研究范围是什么领域？
2、该工作解决了什么问题？
3、介绍问题的已有研究背景？
4、具体来说该工作使用了什么方法？
5、该工作实验效果如何？有什么发现？
6、总体来说，该工作相比已有工作，其创新点在那里？

以markdown格式回答，每个问题对应一个二级标题，并在标题后进行回答。请认真完成该工作，这对你的职业生涯和学术声望至关重要！如果你的回答令我满意，我将给予你丰厚的奖励。
```

1. # 写论文精细打磨套路

## 2.1 Abstract写法

```Markdown
原则
     1. 前半部分，intro里一段一般是abs里一句
     2. 后半部分，和intro的in this paper段表达内容一样 但更简洁
     3. 每个词都要精选，每个词是否存在都需要衡量（删掉后影响意思吗，不影响就删）
     4. 尽量不要用长词（demonstrate -> show，denote -> is）
     5. 不要有太长的任务背景介绍和实验结果分析，重点是in this paper前一句和后一/两句。
     6. 虽然看起来abs是intro的缩写，但不能只缩写，读abs本身也要逻辑顺畅，前一句和下一句的衔接关系强。所以建议不要直接对着intro一句一句翻译，而是看了intro根据自己的模型的理解，自己从头写abs
第一句：讲任务：intro第一段的主要思想
     1. 介绍任务/主流做法/简单讲背景，通过最简洁的方法从背景落到具体任务上，也可以提一下主流方法
     2. 有时候如果你的任务大家都熟悉，不用第一段改也可以，可以直接用第二段改，直接讲任务方法
第二句：讲现有方法的概况介绍及其缺点/问题 intro第二段的主要思想
     1. Intro第二段会讲 intro第一段提到的任务中，现有做法分几类/主要思想
     2. 如果intro第二段写的规范，一般就是第一句（分几类/主要思想）+最后一句（存在问题）
     3. 模板例子
             1. Existing studies focus on xxx-based and xxx-based （分类） methods that xxx（主要思路）, which misses to consider xxx / ignores xxx（缺点）
             2. Existing studies mainly xxx（主要思路）, which misses to consider xxx / ignores xxx（缺点）
             3. Existing studies focus on xxx-based and xxx-based （分类） methods, which misses to consider xxx / ignores xxx（缺点）
             4. 第二段内部可能会分为两三种方法，如果是并列关系好说，用上述模板。如果是递进关系，这里篇幅不一定能让你展开递进，也可以用并列的模板写（上述模板）。
             5. 如果第二段和第三段是并列关系，也可以把第三段的内容AA和第二段BB混在一起讲（Existing studies focus on BB-based and AA-based methods）
第三句：讲更细分的现有方法介绍及其缺点/问题  intro第三段的主要思想
     1. 相比第二句更加聚焦，这部分可能涉及不是一个领域而是一个思路的个别几篇工作
     2. 如果篇幅不够，第三句也可以合入第二句，一起说
     3. 这句可能会说三件事：这流派的工作 分几类/主要思想，有时候会提一下能如何解决第二句中存在的问题（不提也行，提了会更流畅），这类做法存在的问题（常常不在这里讲，而是会单独作为第四句来讲）
     4. 模板例子同上
     5. 如果intro有四段都在讲已有工作，可能逻辑会过长，abstract会放不下，那可以把intro第二、三、四段整合一下，用第二句、第三句讲清楚
第四句：
     1. 一般会讲第三句中提到的方法的缺点，也是本文所针对的核心问题核心motivation。
     2. 一般会用一整句把完整的逻辑讲出来。
     3. 可能参考in this paper段之前的最后一段的最后一两句，基于那个缩写改写，不照抄。
     4. 需要把问题讲透彻，reasonable and convincing
     5. 模板
             1. However, the above methods hardly/miss to/ignore xxxx（缺点漏洞问题），resulting in xxxx（落到我们的任务上会导致什么后果）。
             2. However, the above methods hardly/miss to/ignore xxxx（缺点漏洞问题）。
第五句：
     1. 本文中心句
     2. 来自in this paper第一句的改写缩写，或in this paper第一第二句的改写缩写
     3. 写的时候注意要字里行间呼应第四句提出的问题
第六句：
     1. 本文中心句的补充说明
     2. 或者解释为什么第五句提出的方法，可以看看第四句提出的问题（这有时候也可以合入第五句）
     3. 这一句往往因为篇幅不够，就不要了，合入前一句
     4. in this paper第二句的改写缩写
     5. 写的时候注意要字里行间呼应第四句提出的问题、第五句
第七八九句（普通写法）
     1. 介绍亮点的技术，侧重创新性
     2. in this paper里对每个模块介绍的缩写
     3. 不一定每个模块都介绍，重点介绍（1）有亮点的技术（2）和核心思想呼应的很好的技术（四五六句）
     4. 这部分不一定能写三句，有时候是两句
第七八九句（高级写法）
     1. 不讲具体的技术，全用来解释第四五句的核心思想，不建议这样使用
最后一句
    1. 实验
    2. Intro最后一句的改写缩写
```

## 2.2 Introduction 的 In this paper 段写法

```Markdown
时间顺序安排和思路：时间允许，等 intro 和 method 确定后，依次写 Introduction 的 In this paper 段、abstract、contribution 等。重要性是 abstract > In this paper 段 >= contribution >> conclusion，写作难度是 abstract > In this paper 段 > contribution >> conclusion。Abstract、contribution、conclusion 的信息很多来自于 In this paper 段信息的改写、提炼、启发，但是不能照抄 In this paper 段原话。
思路
和 method 的 overview 部分不一样，method overview 注重整体性好读懂，每个功能都要涉及到，各个功能连起来就能实现任务（例如：模块 1 建图，模块 2 基于图推理，模块 3 推理和 LLM 结合，模块 4 生成）。
In this paper 段注重创新性，需要看着高级、难。
模板
第一句中心句
作为全文中心句，涵盖本文方向相关的限定词（任务名、LLM-based、DP-based、考虑上下文的、考虑历史信息的、task-oriented）。
负责本文的创新点，几个最能体现创新性的词。
需要是本文的中心句，而不是对一个领域的概括。
错误示范：In this paper, we propose a LLM-based rumor detection method considering the xx information via in-context learning.
第二句可能对中心句解释。
负责本文的创新点（或其直观理解、详细解释）。
如果第一句信息量太足，对关键的词，没有详细 / 直观的解释，这里可以解释。
Specifically （或类似的连接词）
每一句 / 两句一个点，介绍重要（亮点）模块
To xxx, we propose/construct/employ/build xxx（方法 / 模块等）, which/that/to xxx（具体做的事情等）.
To xxx，we propose/construct/employ/build xxx（方法 / 模块等） to xxx（具体做的事情）, which/that/to xxx（好处等）.
We propose/construct/employ/build xxx（方法 / 模块等）, which/that/to xxx（具体做的事情等）.
We propose/construct/employ/build xxx（方法 / 模块等） to xxx（具体做的事情）, which/that/to xxx（好处等）.
比如：模块 1 建图，模块 2 基于图推理，模块 3 推理和 LLM 结合，模块 4 生成。
写作原则
一般每个模块 1~2 句，共 3~6 句。根据需要分配详略，创新性强的详细说。
创新性弱的模块，少说或不说。
完成没创新性的申请（操作流程性步骤上），尽量不说。
从操作流程层面，尽量完整，但不完整也没事，以创新点为主。
实验结果的总结（一般一句）
一句话说清楚结论：我们比 baseline 好，如果有篇幅有亮点，可以强调亮点和贡献（sota、提升很大具体提升多少、多少的数据集 / 任务上提升）。
和 abstract 最后一句、contribution 最后一点内容差不多，可以改写，但是不能照抄 。
```

## 2.3 Introduction贡献点写法

```Markdown
任务 / 应用角度解决的问题
     - 从某些角度 / 侧面研究，该应用任务之前无类似侧面，内容大概率来自 motivation 部分。基于 motivation 改写，重点强调自己的工作，避免提及他人缺点。
     - motivation 部分通常在 “in this paper” 段之前，阐述与最相似工作相比的缺点和自身优点，这些优点可作为应用任务角度的贡献。例如，在个性化对话生成任务中，考虑新用户历史行为和用户间交互信息 ；在基于差分隐私文本清洗任务中，采样替换词时考虑词语空间的密度信息，对 xx 进行定制化处理，节约隐私损失。
     - 若为首次提出的内容，可以说明，但风险较大，一般避免。
     - 强调 “for xx tasks”。
亮点模块、亮点方法
     - 若有多个亮点，可列举。
     - 若有理论推导 / 理论结论 / 理论证实 xxx，可单独作为一点，常见于差分隐私、博弈相关论文。
     - 可能的模板：             - We propose AAAA to BBBB, which CCCC
             - AAAA：模块方法
             - BBBB：具体做法
             - CCCC：好处（略写）
实验效果
     - 强调亮点和贡献（如 sota、具体提升幅度、在多少数据集 / 任务上提升 ）
```

## 2.4 Related work写法

```Markdown
与 introduction 里介绍相关工作的区别
比 introduction 里介绍的工作范围更广。
不能和 introduction 重复太多。
最好别和 introduction 分类完全一样，最好有 introduction 之外的东西。
Introduction 是一条线（一个漏斗），related work 是几个面。
篇幅大概 0.75~1 页。
每个工作一句话介绍（最多 2 句）。
一般 3 部分（根据情况 2~4 也行），每部分一个方面。
每个方面可能细分 2~3 种流派的做法，如果内容不多，也可以不细分。
言之有物，不能讲空话和泛泛的描述，要讲具体的工作。
引用的工作要够多，一共 25~40 篇（具体情况可多可少），每个方面引用的工作量均匀，不要出现大段话没有引用，只有白话。
尽量多点新引用：今年 / 去年的要足够，以近三年的为主，尤其针对引用中能看出年份的论文模板。
不要把对手说的太弱，要给人感觉是你击败了比较有实力的对手（但不是特别厉害），intro 里也是。
Related work 里讨论的方法，尤其是最新、听起来最好、递进关系中 “食物链顶端” 的方法。
语法格式
    1. 注意引用
```

## 2.6 Conclusion写法

```Markdown
原则
Conclusion 和本文的 abstract 部分比较像，但内容相似，角度差异较大，不能照搬；
Conclusion 从 “后验” 的角度出发，而 abstract/intro 是从阅读本文之前的角度。
Conclusion 侧重讲解我们提出的东西，背景和 motivation 基本不提。
比 abstract 短很多，若为单栏，最短 5 行，最多 10 行。
不需要太多 abstract、introduction 中花哨的高级词汇和表达。因为 method 部分会讲解这些高级词汇的具体化内容，这里适当提一下具体化的结果即可。
对方法的讲解不用特别细，说明 “我们提出了 xxx，达到 xx 效果” 就行。
对实验的讲解可以长一点，包括有意思的结果体现的事情，说明模型达到的能力。
不一定要写未来展望。
模板
第一句中心句：In summary/in this paper, we proposed xxx for xxtasks.
第二 / 三 / 四句：说一下我们提出的一些具体的技术 / 模块，可以借鉴 method 的 overview+intro 的 in this paper 段。
实验效果证实 xxxx。
```
